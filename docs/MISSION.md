# Mission

PenLock exists to make physical access control personal and unforgeable.

A key can be copied. A PIN can be observed. A static signature image can be photographed. But the way a person draws their signature — the rhythm, the speed, the pressure, the order of strokes — is a behavioral biometric that is uniquely theirs and nearly impossible to replay.

PenLock captures that behavioral fingerprint, verifies it in the cloud, and opens a physical lock in real time.

---

## Core Principles

**Behavioral, not visual.**
Verification is based on how a signature is drawn, not what it looks like. Two signatures that appear identical to the eye will fail if their temporal dynamics differ.

**Real-time.**
The path from signature to unlocked door is measured in milliseconds, not seconds. The system is designed for immediate, low-latency response.

**Minimal attack surface.**
No passwords. No keys. No tokens stored on device. The baseline lives in the cloud. The lock only opens on an explicit positive verification — never on failure or ambiguity.

**Hardware-grounded.**
Verification is not just a software flag. It physically actuates a lock via Arduino, making the security boundary tangible and auditable.

---

## What PenLock Is Not

- Not a general biometric platform
- Not a remote-access or network authentication tool
- Not a replacement for multi-factor systems in high-security contexts
- Not designed for mass deployment in its current form
